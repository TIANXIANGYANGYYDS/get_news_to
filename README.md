# daily_pe_reporter

一个面向 **A 股资讯监控与投研辅助** 的自动化项目。

项目主要用于抓取盘前/盘中资讯，结合 LLM 做结构化分析，落库保存，并通过飞书卡片进行通知，方便快速搭建一个可持续运行的轻量级资讯监控与推送系统。

---

## 项目简介

`daily_pe_reporter` 的目标不是做一个单纯的新闻爬虫，而是构建一个完整的 **“抓取 → 分析 → 入库 → 通知”** 流程，用于服务 A 股盘前和盘中信息处理场景。

当前项目已经覆盖或预留了以下能力：

- 财联社电报抓取
- 启动时自动同步最新数据
- 定时轮询增量抓取
- 使用 LLM 对单条资讯做结构化分析
- 将原始数据与分析结果一起写入 MongoDB
- 通过飞书机器人发送通知卡片
- 支持后续扩展到盘前晨报、复盘、每日汇总等模块

这个项目适合作为以下场景的基础框架：

- 盘前资讯助手
- 舆情监控/资讯监控系统
- A 股题材跟踪系统
- 飞书群投递机器人
- LLM 驱动的结构化资讯处理服务

---

## 核心功能

### 1. 财联社电报自动抓取
系统支持从财联社源拉取最新电报，并进行增量同步。

特性包括：

- 启动时自动拉取一次最新数据
- 运行中按固定周期轮询
- 只处理新增电报，避免重复写入
- 支持按 `event_id` 做唯一去重

---

### 2. 单条资讯 LLM 分析
每条即将入库的资讯会先经过 LLM 分析，输出统一结构的结果。

当前分析结果包括：

- `score`：资讯对 A 股交易层面的影响评分，范围 `-100 ~ 100`
- `reason`：评分原因
- `companies`：涉及公司，没有则为 `NONE`
- `sectors`：涉及板块，没有则为 `NONE`

这样做的价值在于，后续可以直接基于结构化结果做：

- 资讯筛选
- 高价值事件提醒
- 板块热度聚合
- 个股/行业映射
- 飞书卡片展示
- 每日汇总报告

---

### 3. MongoDB 入库
所有资讯会统一写入 MongoDB，原始字段和分析字段同时保留，便于后续检索、统计和再次处理。

典型设计思路：

- 原始资讯字段完整保留
- 在原始数据上增加一个对象字段保存分析结果
- 使用唯一索引确保不重复入库
- 支持按发布时间、主题、事件 ID 等字段检索

---

### 4. 飞书卡片推送
项目集成飞书机器人能力，在数据入库成功后可自动发送卡片到指定群聊。

适合以下场景：

- 新资讯提醒
- 高分资讯推送
- 调试环境验证
- 日常盘前/盘中播报

---

### 5. 定时调度
项目内置调度能力，可用于：

- 每 5 分钟同步一次资讯
- 盘前固定时间发送晨报
- 收盘后生成复盘推送
- 定时执行数据库扫描任务
